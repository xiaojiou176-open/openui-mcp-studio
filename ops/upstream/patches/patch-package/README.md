# patch-package Directory

This directory stores third-party dependency patches generated by `patch-package`.

## Naming Rules

- Patch files are generated by `patch-package` and must not be manually renamed.
- The default format is `<package-name>+<version>.patch`.
- Examples:
  - `lodash+4.17.21.patch`
  - `@scope+pkg+1.2.3.patch`

## Standard Workflow

1. Create a patch
   - Modify the target files under `node_modules/<package-name>`.
   - Run: `npm run patch:create -- <package-name>`

2. Apply a patch
   - Patches are replayed automatically during `postinstall`.
   - You can also run: `npm run patch:apply`

3. Review a patch
   - Explain the patch reason, impact surface, and retirement condition in the PR.
   - Keep the diff minimal and limited to the necessary changes only.

4. Retire a patch
   - Delete the corresponding `.patch` file once upstream ships the equivalent fix.
   - Reinstall dependencies and rerun validation:
     - `npm install --package-lock-only`
     - `npm run lint`
     - `npm run docs:check`
