Workflow at a glance

From brief to reviewed frontend delivery in four visible steps.

OpenUI MCP Studio is designed to keep the whole path legible: generate UI, apply files, run quality gates, and keep evidence for follow-up.

1

Brief

Start with a natural-language request that describes the UI you want to ship.

2

Generate

Turn the brief into React and shadcn-oriented frontend output instead of a loose text blob.

3

Apply

Write files into the target workspace so the workflow can be inspected as real project state.

4

Verify

Use smoke, visual, UI/UX, and release-readiness checks before calling the result good enough.

Default proof target: apps/web
Runtime entry: services/mcp-server
Trust layer: quality gates and evidence
