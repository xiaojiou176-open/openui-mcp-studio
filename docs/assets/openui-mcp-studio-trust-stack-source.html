Why trust the public story

Trust is not a promise here. It is a stack of visible checks.

OpenUI MCP Studio is designed to make “why should I believe this?” easier to answer with proof surfaces, repeatable checks, and public routing.

1. Real proof surface

`apps/web` is treated as the default proof target, so the public story points at a real UI surface rather than a concept sketch.

apps/web as proof target

2. Quality gates stay in the loop

Smoke, visual, UI/UX, and release-readiness checks keep the repository from acting like generated output is automatically done.

Smoke + visual + UI/UX + readiness

3. Public routing is explicit

README, docs, releases, and discussions all tell the same story, so new visitors are not forced to decode the repository from raw file structure alone.

README + docs + release + discussions

4. Governance is visible, not buried

The repo still exposes runtime, contracts, and safety rules, but as trust evidence rather than the first thing a new visitor has to read.

Governance as evidence, not clutter
